Einrichtung — Schritt für Schritt

Befehle gibst du im Terminal ein (Windows: PowerShell oder Eingabeaufforderung · Mac: Terminal).

1
Node.js & Git installieren

Node.js (Version 20 oder neuer, „LTS") von nodejs.org — bringt npm mit. Git von git-scm.com. Beide einfach mit dem Installer durchklicken.

Prüfen, ob es klappt: node -v und git --version sollten je eine Version anzeigen.

2
GitHub-CLI installieren & anmelden

Die gh-CLI von cli.github.com. Danach anmelden — dabei wird dein GitHub-Token automatisch erzeugt, du musst keinen von Hand anlegen:

gh auth login
# → „GitHub.com" → „HTTPS" → „Login with a web browser"
# → den angezeigten Code im Browser eingeben, fertig

Die Skills nutzen gh, um Issues und Repos anzulegen. Mit dieser Anmeldung ist alles abgedeckt — kein separater Token nötig.

3
Claude Code installieren & anmelden
# installieren
npm install -g @anthropic-ai/claude-code

# starten (meldet dich beim ersten Mal mit deinem Anthropic-Account an)
claude

Windows-Stolperstein: Falls claude „nicht gefunden" wird, muss der npm-Pfad in die PATH-Variable. Pfad anzeigen mit npm config get prefix (z. B. %APPDATA%\npm), diesen zu PATH hinzufügen, neues Terminal öffnen.

4
Zugang von Hauke annehmen

Schick Hauke deinen GitHub-Usernamen. Er lädt dich beim Skills-Repo ein — du bekommst eine E-Mail bzw. siehst die Einladung auf GitHub und klickst „Accept".

5
Die Skills holen

Einmalig in deinen globalen Claude-Ordner klonen:

git clone https://github.com/Hauke1979/p2cx-app-factory-skills.git ~/.claude/skills
6
Claude Code neu starten & loslegen

Damit die neuen Befehle erkannt werden. Dann in einem leeren Ordner:

# neuen App-Ordner anlegen und Claude öffnen
mkdir meine-erste-app
cd meine-erste-app
claude

# dann einfach beschreiben, was du bauen willst:
/init Ich will ein kleines Tool bauen, mit dem …
Gut zu wissen
  • GitHub-Token? Brauchst du nicht extra — gh auth login regelt das.
  • Wo liegen die Skills? In ~/.claude/skills — global, also in jedem Projekt verfügbar.
  • Updates? cd ~/.claude/skills && git pull
  • Verirrt? Tippe /help — sagt dir, wo du stehst und was als Nächstes kommt.
  • KI in der App? Dann gibt Hauke dir einen Langdock-Key. Wir nutzen nur Langdock (EU, DSGVO) — nie OpenAI direkt.
  • Live schalten? Machen wir gemeinsam, wenn die App so weit ist. Am Anfang läuft alles lokal.

Das war's an Setup. Ab hier ist Entwickeln vor allem ein Gespräch — die Skills führen dich durch Idee, Spec, Bau, Test und Deploy.
